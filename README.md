📌 Project Overview
This project focuses on designing and implementing a relational database for a hotel management system. The goal is to streamline guest data, room allocations, and payment tracking to provide actionable business insights.

🛠️ Tech Stack
Database: SQL (MySQL / PostgreSQL)

Skills: Relational Database Design, Data Cleaning, Advanced Querying.

📊 Database Architecture
The project utilizes a relational schema with 4 interconnected tables:

Guests: Personal details and contact information.

Rooms: Room categories, availability, and pricing.

Bookings: Guest-to-room mapping, check-in/out dates.

Payments: Transaction details, amounts, and payment status.

🔑 Key SQL Features Implemented
Multi-Table Joins: Used INNER JOIN and LEFT JOIN to consolidate data across all four tables.

Data Integrity: Implemented PRIMARY KEY and FOREIGN KEY constraints to ensure data consistency.

Aggregations: Utilized GROUP BY and SUM/COUNT functions to calculate total revenue and occupancy rates.

Complex Filtering: Written queries to identify peak booking seasons and guest preferences.

💡 Business Insights Derived
Identified the top-performing room categories by revenue.

Analyzed monthly booking trends to optimize staffing.

Tracked payment success rates to identify potential bottlenecks in the checkout process.
